TurbGen.profModels.api module¶

TurbGen mean velocity ‘profile models’ package.

Available profile models¶

log_models.nwtc (alias: log)
The nwtc logarithmic mean wind-speed profile.
log_models.H2O (alias: h2l)
The hydro (water) logarithmic mean velocity profile.
power_models.nwtc (alias: pl)
The nwtc 1/7 power-law mean wind-speed profile.
iec_models.main (alias: iec)
The IEC mean wind-speed profile.
profModelBase
This is the base class for profile models. To create a new profile model, subclass this class or subclass and modify an existing profile model.
profObj
This is the ‘profile object’ class. All profile model __call__ methods must take a TGrun as input and return this class.

Example usage¶

>>> import TurbGen.profModels.api as pm

Create a logarithmic mean wind-speed ‘profile model’ with 18m/s wind at 80m, and surface roughness length of 1m:

>>> my_prof_model=pm.log(18,80,1)

# Now set ‘tgrun’ to use this profile model:

>>> tgrun.prof=my_prof_model(tgrun)

Creating a new profile model¶

Creating new profile models in TurbGen is simple. Simply subclass profModelBase, then:

1) write an __init__ method that takes inputs that define the profile model in general (i.e. irrespective of the spatial grid or other statistics of the flow), and

2) write a __call__ method that takes TGrun as input and returns a profObj, which

For example:

import TurbGen.profModels.api as pm # import the profModels api
from numpy import polyval # Import helper functions.

class my_new_model(pm.profModelBase):
    """
    A polynomial mean-velocity profile, where coefficients can be
    specified at initialization.
    """

    def __init__(self,Uref,Zref,poly_coefs=[1.,0.]):
        # All units should be in meters and seconds.
        self.Uref=Uref
        self.Zref=Zref
        self.poly_coefs=poly_coefs

    def __call__(self, tgrun):
        out=pm.profObj(tgrun)
        # Note here that tgrun contains the 'grid information', and
        # we can use that information to construct the profObj.
        out[0]=self.Uref*numpy.polyval(self.poly_coefs,tgrun.grid.z/Zref)
        # Here we have set the u-component (index 0) to follow the
        # polynomial, and we leave the other components to be zero.
        return out

That’s all that is required to define a new profile model! Now, assuming you have already created a TGrun instance (e.g. tsr) you set that TGrun to use your new model by simply doing, for example:

>>> tgr.prof = my_new_model(3.,20,[.5,0])

Now your TurbGen run will utilize your newly defined model!

Notes¶

For a description of the difference between ‘profile models’ (e.g. ‘my_prof_model’ in example above) and the profile they output (tgrun.prof), see the Code Framework section of the TurbGen documentation.

class TurbGen.profModels.log.nwtc(URef, ZRef, Z0, Ri=0, turbmodel=None)[source]¶

Bases: TurbGen.profModels.base.profModelBase

NWTC logarithmic wind-speed profile model.

Parameters:
URef : float

Reference velocity for the wind profile [m/s].

ZRef : float

Reference height of the reference velocity [m].

Z0 : float

Surface roughness length [m].

Ri : float

The Richardon number [non-dimensional].

turbmodel : str, optional

The name of the turbulence model in this simulationm.

Notes

The exact form of this model is,

\[\bar{U}(z) = U_{Ref}\frac{ln( z / Z0 ) - \psi_M}{ln( Z_{Ref} / Z0) - \psi_M}\]

Where psi_M is a function of Ri, the Richardson number (psi_M=0 for Ri=0), and the turbulence model.

Module contents¶

This is the TurbGen mean velocity ‘profile models’ package. This package contains pre-defined profile models that can be implemented in a TurbGen run.

For more information and to use this module import the profModels.api package, e.g.

>>>    import TurbGen.profModels.api as pm
